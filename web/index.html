Check XML and XHTML documents for Well-Formedness and Validity while editing them in TextMate with support for DTD, W3C XML Schema, RELAX NG and XInclude.

To install, quit TextMate, download TeXMLMate PlugIn and unzip. Then drag the TeXMLMate.tmplugin file to ~/Library/Application Support/TextMate/PlugIns. You may need to create this folder. Launch TextMate and select Show TeXMLMate Palette from the Window menu.

The TeXMLMate PlugIn adds an XML parsing palette to the popular TextMate text editor for Mac OS X. While editing an XML (or XHTML) document in TextMate, you can open the TeXMLMate palette to conveniently check your document for well-formedness or validity against a DTD, W3C XML Schema, or RELAX NG schema. Here's how:

  1. Install TeXMLMate PlugIn.
  2. Begin editing an XML in TextMate (TeXMLMate will always try to use your frontmost TextMate document as the XML document to be parsed).
  3. From the Window menu, select Show TeXMLMate palette.
  4. In the TeXMLMate palette, you can select your Validation options. Select None to just parse your XML document and check it for well-formedness. If you would like to validate your document, first select your preferred schema language.
  5. Then, select your schema document by clicking Browse and selecting from the filesystem. Alternatively, if you'd just like to validate your document against a DTD referenced from your source document's doctype, select DTD and leave the text field blank.
  6. Select other XML parser options by expanding the Settings disclosure triangle.
  7. Click the Parse button.
  8. The top half of the split view will show the results the parsing/validation. Selecting the Verbose checkbox will show more detailed information here.
  9. The bottom half of the split view will show the text representation of the XML parser's output (including defaulted DTD attributes, entity expansions, CDATA merges, etc. according to your parse settings -- this is sometimes helpful for debugging as you can see what the XML parser 'sees'.)
  10. XML Error messages appear in red and warnings appear in yellow. Clicking on either of this will open the offending document (source or schema) in TextMate with the problematic text selected.

TeXMLMate is powered by libxml. Many more features are planned for the future.
